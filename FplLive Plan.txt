========================
6. LIVE POINTS CALCULATION
========================

Do not calculate raw football points manually in MVP.

Use FPL live endpoint player `total_points`.

Algorithm:

For each manager:
1. Get manager picks for the event.
2. Get live player points for the event.
3. Apply multiplier for each pick.
4. Deduct transfer cost.
5. Add to previous official total.

Formula:

LiveGwPointsBeforeHits =
SUM(playerLiveTotalPoints * pickMultiplier)

LiveGwPointsAfterHits =
LiveGwPointsBeforeHits - TransferCost

LiveSeasonTotal =
PreviousTotalBeforeCurrentGameweek + LiveGwPointsAfterHits

Important:
- Use official pick multiplier when reliable.
- If official FPL has not yet processed captain/auto-sub logic, use our projection calculators.
- Store both raw and projected values where helpful.

========================
7. CAPTAIN / VICE-CAPTAIN LOGIC
========================

Implement captaincy projection.

Rules:
- If captain plays any minutes in the Gameweek, captain gets double points.
- If captain plays 0 minutes after all his possible fixtures are finished, vice-captain gets double points.
- If neither captain nor vice-captain plays, no player receives double points.
- For Triple Captain, multiplier is 3 instead of 2.
- If captain does not play and Triple Captain chip is active, apply triple multiplier to vice-captain if FPL rules/data support that behavior.

Important:
- Be careful in double gameweeks.
- A player with 0 minutes is not considered final until all fixtures for his team in the Gameweek are finished.
- During live matches, mark captaincy as “projected” if not final.

========================
8. AUTO-SUB CALCULATION
========================

Implement auto-sub projection.

Formation constraints:
- Exactly 1 goalkeeper
- At least 3 defenders
- At least 2 midfielders
- At least 1 forward
- Total starters = 11

Pick position assumptions:
- Positions 1-11 are starting XI
- Position 12 is bench goalkeeper
- Positions 13-15 are outfield bench players in priority order

Algorithm:
1. Start with positions 1-11.
2. Identify starters with 0 minutes and no remaining fixture in the gameweek.
3. For goalkeeper:
   - Replace only with bench goalkeeper if bench goalkeeper played.
4. For outfield player:
   - Try bench outfield players in bench order.
   - Bench player must have played.
   - Replacement must not break formation constraints.
5. Apply each bench player only once.
6. Return:
   - Final projected XI
   - Substitutions made
   - Players not subbed due to invalid formation
   - Whether projection is final or still pending

Edge cases:
- Double gameweeks
- Players with postponed matches
- Bench Boost active
- Captain not playing and being auto-subbed
- Vice-captain not playing
- Formation validity after multiple missing starters

========================
11. EFFECTIVE OWNERSHIP
========================

Implement effective ownership inside selected mini-league.

Formula:

EffectiveOwnership(player) =
SUM(player multiplier across managers) / manager count * 100

Example:
- Owned normally by 60 managers = 60
- Captained by 20 managers = 40
- Triple captained by 5 managers = 15
- Total managers = 100
- EO = 115%

For each relevant player, calculate:
- Ownership %
- Captaincy %
- Effective ownership %
- User multiplier
- Rank impact per point

RankImpactPerPoint =
UserMultiplier - EffectiveOwnership / 100

Show explanations:
- If user owns/captains a high EO player, points may protect rank.
- If user does not own a high EO player, that player scoring hurts rank.
- If user owns a low EO differential, points may improve rank.

========================
19. IMPORTANT EDGE CASES
========================

Handle these carefully:

1. Captain has 0 minutes but still has another fixture in double gameweek.
Result:
- Do not finalize vice-captain yet.
- Mark as pending.

2. Starter defender did not play and starting formation has only 3 defenders.
Result:
- Cannot replace him with midfielder/forward if that leaves only 2 defenders.
- Must find eligible defender from bench.

3. Bench Boost active.
Result:
- Do not auto-sub normally for points calculation because all 15 count.
- Still show normal bench information.

4. Transfer hit.
Result:
- Deduct event transfer cost from live GW score.

5. League with many managers.
Result:
- Use pagination, batching, cache, and background job progress.
- Do not block HTTP request for too long.

6. Event not started.
Result:
- Live points should be 0 or official current state.
- Show “Gameweek not started”.

7. Event finished.
Result:
- Live data should become final.
- Auto-sub/captaincy projections become final if FPL data confirms.
